Qué es el criptosistema de clave pública

La criptografía de clave pública es un criptosistema en el que las claves aparecen en pares. Los cambios realizados con una de las teclas sólo se pueden deshacer con la otra tecla. Una clave (la clave privada) se mantiene secreta y la otra se hace pública.

  • Clave Pública: Esta clave se utiliza para cifrar el material. Cualquier persona que quiera enviar mensajes de forma segura puede acceder a él.
  • Clave Privada: Esta clave se utiliza para descifrar material cifrado mediante la clave pública. Debe ser confidencial y accesible sólo para destinatarios autorizados.

    • Cómo funciona y cuáles son sus algoritmos representativos

    La criptografía de clave pública opera a través del empleo de dos claves que guardan una relación matemática entre sí:

  • la clave pública.
  • la clave privada.

    • Ventajas, desventajas y diferencias con el criptosistema simétrico

    Ventajas:

    • Mayor seguridad se logra al emplear dos claves.
    • No hace falta que compartas la clave privada.
    Desventajas:
    • Más pausado que el criptosistema simétrico.
    • Complicaciones al crear contraseñas.
    Diferencias con el criptosistema simétrico:
    • El sistema criptográfico simétrico emplea una única clave tanto para encriptar como para desencriptar la información.
    • Los criptosistemas simétricos, por lo general, tienden a ser más veloces, aunque disminuye su nivel de seguridad en caso de que la clave sea comprometida.

    Caso práctico de aplicación

    Un ejemplo ilustrativo es la implementación del algoritmo RSA para salvaguardar información en comunicaciones seguras por la web, tal como en la utilización del protocolo HTTPS.

    Observaciones y recomendaciones para implementar el criptosistema

    Observaciones:

    • Es fundamental garantizar que las contraseñas se creen y guarden de forma segura.
    Recomendaciones:
    • Es recomendable emplear bibliotecas y herramientas criptográficas que hayan sido ampliamente probadas, y es aconsejable no desarrollar algoritmos propios si no se cuenta con el conocimiento adecuado.
